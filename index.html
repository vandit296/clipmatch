Rather not cut in the browser? Two other ways:

① Use your own editor — Descript, Premiere, CapCut…

Best if you already edit somewhere. No new tools, no terminal.

  1. Click Download timestamps.json (or just read the table above).
  2. In your editor, move the playhead to a clip's Start time and set an in-point.
  3. Go to its End time, set an out-point, and export that range.
  4. Repeat per clip. Skip any ⚠ low row, or check it by ear first.

② Run the ffmpeg commands yourself

Fastest for a whole batch. The commands tell ffmpeg (a free tool) exactly what to cut — one line per clip.


      

        How do I actually run these? ▸
        

          
Step 1 — install ffmpeg (one time). Mac: open Terminal and run brew install ffmpeg (needs Homebrew from brew.sh). Windows: download it below, unzip, and add it to your PATH.

          
Step 2 — open a terminal. A terminal is a window where you type commands. Mac: open the Terminal app (press ⌘+Space, type “Terminal”, press Enter). Windows: open PowerShell (search it in the Start menu).

          
Step 3 — point the terminal at the folder that holds your video. The cd command (“change directory”) moves it there. Pick whichever is easiest — no dragging needed:

          
    
            
  • Type the path: e.g. cd ~/Downloads/episode — swap in the folder your video lives in.
    • 
            
  • Copy the path from Finder (Mac): hold Option, right-click the folder → Copy “…” as Pathname; then type cd  (with a space) and paste (⌘+V), Enter.
    • 
            
  • Open the terminal already inside the folder: Mac — right-click the folder → New Terminal at Folder (enable it once in System Settings ▸ Keyboard ▸ Shortcuts ▸ Services). Windows — hold Shift, right-click the folder → Open in Terminal. Then there's no cd to type at all.
    • 
          

          
Step 4 — paste the commands (⌘/Ctrl+V) and press Enter. Each line makes one clip in that folder. A line reads: ffmpeg -ss START -i "video" -t LENGTH … "clip.mp4" — i.e. start time, your video, clip length, output name.

How it works: the waveform of each audio clip is matched against the video's audio track (normalized cross-correlation) to find the exact moment, then ffmpeg.wasm cuts the video right here in the page. No ffmpeg install, no terminal — the tradeoff is the ~30 MB engine load and browser memory limits on very large videos.
